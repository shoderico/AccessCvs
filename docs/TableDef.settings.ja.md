# TableDef.settings 仕様

このファイルは、Access の通常テーブル（ローカルテーブル）に対して、**データ出力の対象**を指定する設定ファイルです。

## ファイルの場所

プロジェクトのルートディレクトリに配置します。

```
<プロジェクトルート>/
├── TableDef.settings   ← このファイル
├── tabledefs/          ← 出力先フォルダ
└── ...
```

## 書式

UTF-8（BOM なし）、改行コードは CRLF（`\r\n`）です。

キーと値は `key =value` の形式で記述します。複数のテーブルを指定する場合は、`Begin TableData ... End` のブロックで囲みます。

```
Begin TableData
    TableName =T_Customers
    TableName =T_Orders
End
```




## 設定項目

### TableData セクション

データ出力の対象とするテーブル名を指定します。

| キー        | 意味                                           |
|-------------|------------------------------------------------|
| `TableName` | データを出力するテーブルの名前                 |

- 1 つの `TableData` ブロック内に、複数の `TableName` を記述できます。
- テーブル名は大文字・小文字を区別します（CaseSensitive）。




## デフォルト動作

この設定ファイルが存在しない場合、または `TableData` セクションが空の場合は、**データ出力は行われません**。

テーブル構造（`.xml`）は常に CVS 管理の対象となりますが、**テーブルデータ（`.dat`）は明示的に指定したテーブルにのみ出力**されます。




## 出力先フォルダ

**`tabledefs/`** フォルダに出力されます。




## 出力される情報

通常テーブルごとに以下のファイルが出力されます。

### ファイルの種類と拡張子

| ファイル種別     | 拡張子   | 内容                                           | 出力条件                     |
|------------------|----------|------------------------------------------------|------------------------------|
| テーブル構造     | `.xml`   | テーブル構造（フィールド定義、インデックス、プロパティなど） | 常に          |
| テーブルデータ   | `.dat`   | テーブルデータ（XML形式）                    | `TableData` セクションで指定されたテーブルのみ |


### ファイル名例
```
T_Customers.xml
T_Customers.dat  ← TableData で指定しているので、データが出力される
T_Items.xml      ← TableData で指定していないので、データが出力されない
T_Orders.xml
T_Orders.dat     ← TableData で指定しているので、データが出力される
```


### テーブル構造（.xml）とデータ（.dat）

- `.xml` は `ExportXML` で出力され、`TableDefSanitizer` でサニタイズされます。
- `.dat` は `TableData` セクションで指定されたテーブルに対してのみ出力されます。
- **構造（.xml）は Export のみ**で、Import 時には使用されません（Export-only）。

**UI オープン後の HasData チェックボックス操作について**:
UI を開いた時点で `refreshItems` により TempDir への初期エクスポートが行われます。
その後、HasData 列のチェックボックスをオンにした場合、Export ボタン押下時に `ensureDataInTempDir` が自動で呼ばれ、該当テーブルのデータ（.dat / .dattmp）が TempDir に補完されます。
その後、通常の Copy 処理により SourceDir（`tabledefs/`）へコピーされます。
構造ファイル（.xml）の再出力は行われません（データ部分のみ）。

チェックをオフにした場合（UI オープン時に hasData オンだった InBoth アイテムを後からオフにしたケース）:
Export ボタン押下時に `removeDataFromSourceDir` が呼ばれ、SourceDir（`tabledefs/`）の該当 .dat のみが削除されます。
TempDir の .dat / .dattmp は削除されません（ユーザ選択: leave stale）。
構造ファイル（.xml）は一切更新されません（データ部分のみのオフ）。
（UI オープン時に hasData オフだったアイテムや、InSourceDirOnly のアイテムについては従来通りの動作です。）










## CVS からインポートしたときの動作

CVS からプロジェクトに取り込む（インポートする）ときに、以下のような処理が行われます：

1. プロジェクト内に同じ名前のテーブルがすでに存在する場合、一度削除されます。
2. `.xml` ファイルからテーブル構造が復元されます（フィールド定義やインデックスなどが再作成されます）。
3. 同じ名前の `.dat` ファイルが存在し、かつそのテーブルが `TableData` セクションで指定されている場合、データが追記されます。



## OdbcTable.settings との違い

| 項目               | TableDefProcessor（通常テーブル） | OdbcTableProcessor（ODBC リンク） |
|--------------------|-----------------------------------|-----------------------------------|
| デフォルトで処理   | する                              | **しない**                        |
| 設定ファイル       | `TableDef.settings`               | `OdbcTable.settings`              |
| 出力フォルダ       | `tabledefs/`                      | `odbctables/`                     |
| 出力形式           | `.xml` + `.dat`                   | `.odbc` + `.xml` + `.dat`        |
| テーブル構造出力   | 可能                              | 可能（Export のみ）               |
| データ出力         | 可能（`.dat`、TableData 指定時） | 可能（`.dat`、TableData 指定時） |
| データ出力対象指定（UI） | HasData列のチェックボックスで指定 | HasData列のチェックボックスで指定（対応済み） |
| 構造の Import      | 可能                              | 不可（Export-only）               |

## 注意点

- `TableDef.settings` で指定するのは「データ出力の対象」のみです。テーブル構造自体は常に CVS 管理対象となります。
- データ出力対象に指定したテーブルは、CVS 管理下でデータの変更履歴を追跡できます。
- 大量のデータを含むテーブルをデータ出力対象にすると、`.dat` ファイルが非常に大きくなる可能性があります。必要最小限のテーブルに絞って指定することをおすすめします。
- ODBC リンクテーブルと異なり、通常テーブルはローカルにデータを持つため、CVS で構造とデータの両方を管理するのに適しています。
